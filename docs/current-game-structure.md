# CHU CHASE Current Game Structure

## ゲーム概要

CHU CHASE は、2人対戦の3D鬼ごっこゲーム。

片方が `CHUSER`、もう片方が `DODGER` になり、1分間のラウンドで遊ぶ。

- `DODGER` は逃げ切れば勝ち
- `CHUSER` は1回でも捕まえれば勝ち

---

## 基本フロー

1. 片方がルームを作成
2. もう片方がルームコードで参加
3. 2人揃うと役割が自動決定
4. 各自 `START READY`
5. ラウンド開始
6. 最初の5秒は `DODGER` だけ移動可能
7. 5秒後に `CHUSER` も移動開始
8. 捕獲 or タイムアップで終了
9. `YOU WIN! / YOU LOSE!`
10. `RESTART` で再戦

---

## 画面構成

### 1. ルーム作成 / 参加画面

- `Chu Chase`
- `CREATE ROOM`
- ルームコード入力
- `JOIN`
- スマホ縦持ち想定
- `CONNECTED` は右上

### 2. ルーム待機画面

- 作成したルームコードを表示
- 相手が入るまで待機
- `BACK` で戻れる
- スマホ縦持ち想定
- `CONNECTED` は右上

### 3. ゲーム準備画面

- 2人参加後
- 3Dゲーム画面へ遷移
- 最初はキャラが後ろ向きで上からポトンと落ちてくる
- 着地後に少し溜めてから、発表前SEに合わせてくるっと正面へ振り返る
- 振り返り後に役割表示と `START READY` を表示
- 役割表示 `YOUR ROLE / CHUSER or DODGER`
- `START READY`
- `あなたの準備待ち` / `相手待ち`
- スマホは横持ちゲーム想定

### 4. ラウンド開始演出画面

- 中央に `5 / 4 / 3 / 2 / 1` のカウント
- DODGER側は小さく `ESCAPE!!`
- CHUSER側は小さく `HOLD YOUR CHU...`
- CHUSERは5秒間、透明なカプセルに閉じ込められて動けない状態を表現
- 最初の5秒間は `DODGER` だけ動ける
- 表示は小さめのステッカー風

### 5. ゲームプレイ画面

- 3Dフィールドで鬼ごっこ
- `TIME` は画面上部中央
- HUDはかなり最小限
- スマホ操作はタップ位置に出る丸いバーチャルジョイスティック
- PC操作はキーボード / ポインター操作

### 6. 捕獲演出画面

現状は仮実装。

- `BUCHU!!`
- カメラ寄り
- 画面揺れ
- `!?`

今後、以下を強化予定。

- 顔崩壊
- ブチュー感
- ヒットストップ
- キスされた側のリアクション

### 7. リザルト / 再戦画面

- `YOU WIN! / YOU LOSE!`
- `RESTART`
- ルームコードなどは非表示
- シンプル構成

---

## 現在のルール

- 1ラウンド: 60秒
- 捕獲回数: 1回で終了
- 開始直後: 5秒だけ `DODGER` 先行
- `CHUSER` が捕まえたら `CHUSER` 勝利
- `DODGER` が逃げ切ったら `DODGER` 勝利
- ルームは2人制
- 役割は2人揃った時点で自動割り当て

---

## 現在のビジュアル状態

- 白基調
- ニューモーフィズム寄り
- UI / ステージ色は `neumorphism-ui-design-system.md` のDesign Tokens基準
- 3Dキャラは仮の豆型キャラ
- `CHUSER` / `DODGER` は色違い
- 背景は白〜ライトグレー基調のシンプルなステージ
- ニューモーフィズム配色の簡易マーカーあり
- 本格的なキャラデザイン / 背景デザインはまだ未反映

---

## 重要な技術方針

- Three.jsで3D実装
- CSSでcanvas自体は回転しない
- スマホ縦持ちでも、ゲーム中はThree.js側で横画面として表示
- タッチ座標ズレを避けるため、canvas transform回転は使わない
- ルーム / 対戦同期はSocket.IO
- サーバー側で位置更新・捕獲判定・勝敗判定を管理
